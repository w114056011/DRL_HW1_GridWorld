## 1. 初始開發需求 (Initial Prompt)

**目標：** 開發一個大小為 nxn 的網格地圖，允許用戶指定維度 `n`（範圍從 5 到 9），使用 Flask 網頁應用程式。

**功能要求：**
1. 用戶可以透過滑鼠點擊指定起始單元格（點擊後顯示為綠色）和結束單元格（點擊後顯示為紅色）。
2. 用戶可以設定 `n-2` 個障礙物，透過滑鼠點擊將這些單元格變為灰色。

**開發結果：**
- 建立了基本的 Flask 伺服器 `app.py`。
- 建立了前端介面與互動邏輯 `templates/index.html`。
- 狀態轉換邏輯已實作：起點 (綠色) -> 終點 (紅色) -> (n-2)個障礙物 (灰色) -> 設定完成。

## 4. HW1-2 策略顯示與價值評估 (Policy Evaluation)

**使用者提示詞：**
> HW1-2: 策略顯示與價值評估
> 目標：顯示每個單元格的隨機生成行動（上下左右箭頭）作為策略。
> 功能要求：使用策略評估推導出每個狀態的價值 V(s)。
> 效果如圖片所示

**AI 行動與開發結果：**
- 修改了 `templates/index.html`。
- 新增了雙視窗佈局 (Value Matrix 與 Policy Matrix)，並使用藍色格線設計以貼近圖片效果（預設轉為 `n=7` 以對齊圖片中的 7x7）。
- 實作了隨機策略生成器，將上下左右箭頭隨機分派給每一格開放空間，並渲染於畫面。
- 實作了 Iterative Policy Evaluation 演算法（設定 Discount Factor `GAMMA=0.9`、每步懲罰 `R=-1`、抵達終點 `R=+10`），並運用演算法計算出每個狀態在隨機策略下的預期價值 $V(s)$。
- 在畫面上分別將每個狀態的 $V(s)$ (取到小數第二位) 與 Policy 箭頭 (↑, ↓, ←, →) 渲染至兩組對應且獨立的矩陣之中。

## 5. HW1-3 使用價值迭代算法推導最佳政策 (Value Iteration)

**使用者提示詞：**
> HW1-3: 使用價值迭代算法推導最佳政策
> 1.價值迭代算法：
> -實現價值迭代算法來計算最佳政策。
> -使用該算法推導每個格子的最佳行動。
> 2.最佳政策顯示：
> -顯示推導出的最佳政策，通過顯示每個狀態（格子）對應的行動來展示最佳政策。這些行動應該取代之前顯示的隨機行動。
> 3.顯示價值函數：
> -在執行價值迭代後，更新格子以顯示每個狀態的價值函數 V(s)。
> -每個格子應顯示對應的值，表示在最佳政策下該狀態的期望回報。
> 
> 其他注意事項：
> -使用 Flask 作為後端，HTML/CSS 作為前端來顯示格子並處理用戶交互。
> -確保 Flask 應用程序是交互式的，用戶可以點擊格子來設置起始點、終點、障礙物，並查看生成的政策。
> -提供可視化的功能，讓用戶能夠清楚地看到隨機政策、價值函數和最佳政策的變化。
> 效果如圖片所示

**AI 行動與開發結果：**
- 修改了 `templates/index.html`。
- 將原本的 **Policy Evaluation (策略評估)** 邏輯替換為 **Value Iteration (價值迭代)** 演算法。
- 將操作按鈕文字從「執行策略評估」替換為了「執行價值迭代 (Value Iteration)」。
- 背景在計算各個狀態（State）的預期價值時，會遍歷所有的操作 (↑、↓、←、→)，找尋該狀態在下一階段可以獲得的最大價值（Bellman Optimality Equation）。
- 收斂結果完成後，左側矩陣會顯示**最佳價值函數 (Value Function, V(s))**，而右側矩陣會一併抽出**最佳行動 (Optimal Policy, π*(s))** 的箭頭來取代原本的隨機箭頭顯示。

## 6. 後端運算重構 (Refactoring Value Iteration to Backend)

**使用者提示詞：**
> 將Value Iteration寫在後端運算

**AI 行動與開發結果：**
- **後端修改 (`app.py`)**：新增了一支 API Endpoint `/evaluate`。前端在使用者設定完地圖後會將格式（矩陣大小、障礙物位置、與終點位置）發送給這隻 API。所有的 Value Iteration 以及找尋最佳策略路徑 (Optimal Policy) 的迭代運算都會在此利用 Python 執行。
- **前端修改 (`templates/index.html`)**：將之前由瀏覽器端 JS 負責的演算法刪除，改成以非同步 (`async`/`fetch`) 的方式呼叫後端的 API `/evaluate`。當收到後端的結果後，再負責將得到的價值與策略箭頭渲染於畫面上。
- **效益**：確保演算法計算邏輯與頁面渲染抽離，提升了前端的效能，並為未來更複雜的模型運算（如串聯真正的 RL Model）鋪路。
